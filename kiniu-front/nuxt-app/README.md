# 灵枢 AI 工程实验室前端

这是灵枢 AI 工程实验室的前端界面，基于 Nuxt 4 和 Vue 3 构建。界面以课程路线和可验证实验为主，配套提供对话实验、Agent 编排、会话回放和运行设置。

## 功能模块

- **课程路线**：主入口，按课程目录完成工程工件、运行确定性验收并解锁后续实验。
- **对话实验**：接收用户需求并展示下一步动作，用于验证已完成的 Agent 与工作流。
- **编排台**：包含任务流、智能体和导出回放三个子工作区。
- **设置**：配置语言、主题、后端地址、上游接口、密钥和模型名。
- **实验工作台**：同屏呈现任务讲义、工件编辑器、导入入口、验收证据和 Mentor 反馈。

## 本地开发

安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run dev
```

默认访问地址：

```text
http://localhost:3000
```

## 生产构建

```bash
npm run build
```

本地预览生产构建：

```bash
npm run preview
```

## 后端连接

前端默认连接后端地址：

```text
http://localhost:8080
```

可以在设置页修改“实验室后端地址”。前端会使用该地址请求会话推进、任务流、智能体配置和会话导出接口。

学习中心使用以下后端接口：

- GET /learn/catalog 和 GET /learn/progress：加载课程目录与当前进度。
- POST /learn/tasks/{taskId}/check：提交工件并返回逐项证据、得分和解锁后的进度。
- POST /learn/tasks/{taskId}/feedback：基于确定性检查结果请求导师解释。
- POST /learn/tasks/{taskId}/publish-agent：将通过的陪伴 Agent 发布到 student- 命名空间。

## 设计原则

- 不引入额外 UI 组件库，减少前端依赖和打包体积。
- 课程路线与实验工作台优先，对话和 Agent 编排作为验证工具按需展开。
- 借鉴 LabEx Challenge、Course 与 Lab 页的信息架构，用高密度任务状态、课程路径和同屏证据降低实验切换成本。
- 保持中文排版简洁，减少首屏长说明和过度装饰。
- 适配桌面优先的本地工具场景，移动端保证可读可用。
