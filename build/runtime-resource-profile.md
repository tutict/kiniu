# Runtime Resource Profile

This project is tuned first for a single-user local runtime.

## Daily Runtime

- Run the GraalVM native backend as the only long-lived application process.
- Serve the Nuxt app from static output generated by `npm run generate`.
- Do not keep `npm run dev`, `nuxt preview`, or Nuxt devtools running for daily use.

## Backend Limits

- `game.sessions.max-state-sessions` limits hot `WorldState` instances.
- `game.sessions.max-memory-sessions` limits recent dialogue and private agent memory caches.
- `game.sessions.max-cached-archives` limits cached session archive metadata.
- Full session turns are stored as append-only JSONL records and are assembled only when exported.

## Acceptance Targets

- Idle backend CPU should settle near zero.
- Idle backend RSS should remain stable instead of growing with old session IDs.
- A 100-turn session should not rewrite the full session export on each turn.
- Production frontend usage should not require a resident Nuxt server process.
